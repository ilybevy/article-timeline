Formalization of Distribution Segmentation

1. Formalization of your problem

You have an ordered sequence:

$$P_1, P_2, \dots, P_n$$

Each \(P_i\) is a probability distribution over keywords:

$$P_i(w) \ge 0, \quad \sum_w P_i(w) = 1$$

You want to partition indices into contiguous segments:

$$S = \{[a_1,b_1], [a_2,b_2], \dots, [a_k,b_k]\}$$
  • Only adjacent merging allowed
  • Iterative refinement allowed
  • Goal: find best segmentation

2. What “best” must mean (your constraints translated)

You require:

  • (A) Local consistency: Score of a segment must depend only on that segment
  • (B) Merge invariance: Order of merging must not affect final score
  • (C) Recomputability: After merging, representation must be recomputable from base data
  • (D) Unbiased comparison: All segments must be comparable using same functional form

3. Key construction: segment representation must be “re-anchored”

This is crucial. Define segment distribution:

$$P_{a:b}(w) = \frac{\sum_{i=a}^b N_i P_i(w)}{\sum_{i=a}^b N_i}$$

This ensures:

  • Linearity
  • No path dependence
  • Merge = weighted average (associative)

So: merging is mathematically stable (important)

4. You need a single global scoring functional

Define a reference-independent distortion measure.

The correct choice: Jensen–Shannon based energy

We define Segment cost:

$$C(a,b) = \sum_{i=a}^b N_i \cdot JS(P_i \,\|\, P_{a:b})$$

Where:

  • \(P_{a:b}\) is the segment barycenter
  • \(JS\) is Jensen-Shannon divergence

Why JS (not KL)

KL fails your requirements because it is asymmetric, unstable under sparsity, non-bounded, and sensitive to ordering effects. JS ensures:

  • Symmetry
  • Bounded [0, log 2]
  • Stable under merging
  • Interpretable as entropy gain

5. Global objective (what you optimize)

For segmentation \(S\):

$$Score(S) = - \sum_{(a,b)\in S} C(a,b) - \lambda |S|$$
  • First term: internal coherence loss
  • Second term: complexity penalty

6. Merge / split rule (your “binary decision”)

You only allow: merge adjacent segments OR keep them separate. For two adjacent segments A and B:

Step 1: compute costs

  • Separate: \(C(A) + C(B)\)
  • Merged: \(C(A \cup B)\)

Step 2: decision

Merge if: \(C(A \cup B) + \lambda < C(A) + C(B)\). Otherwise split.

7. Why this is order-independent

This is your key requirement. It works because:

  • Property 1 — barycenter associativity: \(P_{(A \cup B) \cup C} = P_{A \cup (B \cup C)}\) (weighted average).
  • Property 2 — JS is computed only from distributions: No dependence on merge history.
  • Property 3 — cost is segment-local: Recomputation does not depend on path.

8. Why repeated passes are safe

You can do iterative merging because each merge only depends on current partition and there is no global memory except aggregated distributions. This is equivalent to hierarchical agglomerative clustering with a well-defined energy functional.

9. Your “recalculation requirement”

After merging A and B: You MUST recompute \(P_{A \cup B}\) from base distributions, NOT from previous approximation.

Implementation rule: Never store approximated distributions only. Always keep base \(P_i\) and segment indices, then recompute exactly.

10. Final algorithm (clean version)

  1. Initialization: Each year is its own segment: \(S=\{[1,1],[2,2],\dots,[n,n]\}\)
  2. Repeat:
    • Compute all adjacent merge costs
    • Find best merge (greedy or DP)
    • Merge if improvement > threshold
    • Recompute segment distributions exactly
    • Repeat until no improvement

You are building a minimum entropy partitioning of a time-ordered sequence of categorical distributions under a JS distortion metric. This is equivalent to change-point detection and MDL segmentation.
